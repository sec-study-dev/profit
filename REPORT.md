# profit — Wave 3 Aggregation Report

This report aggregates the output of the Wave-2 research swarm: 68 Foundry
PoCs documenting *combinations of distinctive DeFi protocol mechanisms* on
Ethereum mainnet. Every PoC lives under `strategies/FXX-NN-<shortname>/`
and is a self-contained fork-replay against a pinned block.

The per-strategy master table is in `strategies/_index.md`.

---

## 1. Executive summary

- **68 strategies** generated by 17 parallel Wave-2 agents across **17
  families** (F01..F17), 4 strategies per family.
- **Type distribution** (Wave-3 classification, see `_index.md` Section A):
  - **Atomic** (single-tx, flashloan-bootstrapped depeg / rate-cache arbs): **22**
  - **Positional** (multi-block carry, leveraged loops, LP positions): **34**
  - **Points-based** (LRT loyalty + EigenLayer + airdrop speculation): **9**
  - **Vote / bribe** (veCRV / vlCVX governance economics): **2**
  - **Liquidation-based**: 1 (F05-02 soft-liquidation harvest; F05-01 and
    F06-01 are also liquidation-adjacent but classified under atomic
    arbitrage because their entry is a one-tx flashloan trade).
- **57 distinct protocols / primitives** referenced across all strategies
  (LSTs, LRTs, MMs, CDPs, AMMs, restake platforms, synth platforms).
- **Recurring primitives.** `eMode` appears in **33** READMEs (Aave / Spark
  variants); `flashloan / flashmint` in **11** atomic strategies (Maker
  DssFlash, Morpho free flash, Balancer Vault 0-fee flash, Aave
  flashLoanSimple, UniV3 flash); ERC4626 wrappers in **15** (sDAI, sUSDS,
  sUSDe, syrupUSDC, MetaMorpho, wOUSD, Spark vaults).

## 2. Methodology

### How strategies were generated

- A **Plan agent (Wave 1)** scaffolded the Foundry skeleton: `foundry.toml`,
  `remappings.txt`, `src/constants/Mainnet.sol`, `src/interfaces/<family>/`,
  `test/utils/StrategyBase.t.sol`, `STRATEGY_IDS.md` (the family ownership
  / collision contract), and a `Makefile` exposing
  `install / build / test / test-one / summary`.
- **17 Wave-2 agents** ran in parallel, each owning one `FXX` row in
  `STRATEGY_IDS.md`. Each agent was responsible for 3-5 PoCs under its
  family namespace, each PoC consisting of:
  - `strategies/FXX-NN-shortname/README.md` (Mechanism, Why-it-composes,
    Preconditions, Strategy steps, PnL math, Block pinned, Risks, Result).
  - `strategies/FXX-NN-shortname/PoC.t.sol` extending `StrategyBase`.
- **Wave 3 (this report)** consumes the strategy tree, normalises the
  outputs, and produces this REPORT + `strategies/_index.md`.

### The PoC pattern

Every PoC follows the same shape:

```solidity
contract FXX_NN_PoC is StrategyBase {
    uint256 constant FORK_BLOCK = <pinned>;

    function setUp() public override {
        vm.createSelectFork(getMainnetRpc(), FORK_BLOCK);
        super.setUp();
    }

    function test_strategy() public {
        _startPnL("FXX-NN");
        // ... mechanism interactions ...
        _endPnL("FXX-NN");
    }
}
```

`_startPnL` snapshots the strategy contract's USD-denominated balances and
gas counter; `_endPnL` prints the result block that downstream tooling
will grep:

```
==== STRATEGY FXX-NN ====
pnl_usd=<int256, 6 decimals>
gas_usd=<uint256, 6 decimals>
net_usd=<int256, 6 decimals>
========================
```

### Status taxonomy

- **theoretical** — mechanism constructed correctly against on-chain ABIs,
  no fork replay performed (most common).
- **theoretical-historical-replay** — parameterised to a specific past
  event (SVB weekend, Renzo depeg, June 2022 stETH discount); realising
  the printed PnL requires archive RPC at that block.
- **mechanically-reproducible** — on-chain calls themselves exercised
  end-to-end on the pinned fork; *PnL accrual* leg requires `vm.warp`.
- **empirically-validated** — both legs exercised on-fork *and* strategy
  asserts a strict positive PnL at the pinned block.

### Caveats

- **`forge` is not installed in the Wave-3 build environment.** No PoC
  was compiled or executed. Statuses are Wave-2 self-reports.
  Verification belongs to the user (§5).
- Wave-2 agents did **not** coordinate on the exact rendering of the
  Result block. `_index.md` preserves text verbatim; the top-N tables
  normalise via documented heuristics.
- A non-trivial fraction of strategies have **`TODO verify`** markers on
  addresses (Liquity v2 BOLD, Symbiotic vaults, Karak vaults, certain
  Pendle market ids, certain Curve pool addresses for newer stables).
  Enumerated in §6.

## 3. Family-by-family findings

### F01 — LST looping
**Mechanism:** Recursive supply-borrow-swap loop of an LST against WETH
using Aave / Morpho eMode (LTV up to 0.945 in Morpho's wstETH/WETH market).
**Coverage:** wstETH-on-Aave (F01-01), wstETH-on-Morpho (F01-02),
rETH-on-Aave (F01-03), cbETH historical-regime replay (F01-04).
**Insight:** F01-02 (Morpho single-tx bootstrap) is materially superior to
F01-01 net of gas because the loop opens atomically rather than iteratively.
**Most likely empirically profitable:** **F01-02** at a current block.

### F02 — LRT looping & restake
**Mechanism:** Same loop as F01 but on an LRT, where the real alpha is the
multi-protocol point stack (EtherFi + EigenLayer + LRT-native + airdrops).
**Coverage:** weETH-Morpho-flash (F02-01), ezETH-Pendle-YT point-decoupling
(F02-02), pufETH stacked re-hypothecation (F02-03), weETH-Aave-eMode (F02-04).
**Insight:** F02-02's point-decoupling via Pendle YT is a *qualitatively
different* trade — it converts a leverage trade into an option on point prices.
**Most likely empirically profitable:** **F02-01** for the cash leg;
F02-02 / F02-03 dominate in any realistic point realisation but alpha is off-chain.

### F03 — LST/LRT basis & peg
**Mechanism:** Atomic flashloan-funded arb between an AMM (Curve / Balancer)
and the LST's redeem path or canonical rate provider.
**Coverage:** stETH/ETH Curve depeg + Lido withdrawal queue (F03-01), ezETH
Balancer depeg around the Renzo April 2024 event (F03-02), rETH Balancer
rate-provider lag (F03-03), multi-LST triangular (F03-04).
**Insight:** F03-02 is the corpus's highest-magnitude single-event arb (+$10-90k
per 200 WETH).
**Most likely empirically profitable:** **F03-02** at the pinned block.

### F04 — Maker DSR / sDAI / sUSDS
**Mechanism:** DssFlash (free DAI mint) + zero-fee PSM (USDC↔DAI 1:1) +
sDAI/sUSDS/Spark borrow loops.
**Coverage:** SVB-weekend USDC depeg arb (F04-01), sDAI Spark collateral (F04-02),
sUSDS leveraged via DAI (F04-03), PSM + Aave USDC supply spike (F04-04).
**Insight:** F04-04 is the clearest demonstration that the Maker primitive stack
supports a *structurally fee-free* loop — every leg has zero fee.
**Most likely empirically profitable:** **F04-01** at the SVB block.

### F05 — crvUSD LLAMMA
**Mechanism:** Curve's LLAMMA AMM uses an EMA-lagged `price_oracle()` — during
fast moves, a searcher can buy in-band collateral cheaper than spot.
**Coverage:** wstETH band-cross (F05-01), WBTC soft-liq harvest (F05-02), wstETH
leveraged borrow (F05-03), crvUSD peg via DssFlash + Curve (F05-04).
**Insight:** F05-01/02 are the most-MEV-flavored — they live or die on private
orderflow.
**Most likely empirically profitable:** **F05-04** at the Oct 2023 launch-period
block (wide premium, empty searcher field).

### F06 — Liquity v1 / v2
**Mechanism:** v1: LUSD redemption arb funded by DssFlash; SP yield + ETH gain
compounding. v2: BOLD redemption against the lowest-interest-rate trove.
**Coverage:** v1 redemption (F06-01), v1 SP loop (F06-02), v2 redemption sniper
(F06-03), v2 leveraged BOLD borrow (F06-04).
**Insight:** v2 strategies are gated on BOLD mainnet addresses
(`Mainnet.BOLD == address(0)` currently) and ship as theoretical placeholders;
v1 strategies are fully reproducible.
**Most likely empirically profitable:** **F06-01** at the LUSD-3pool discount
blocks.

### F07 — Pendle PT / YT
**Mechanism:** PT sold at a discount to underlying; looping it on Morpho captures
pull-to-par. YT decouples the points stream from the principal.
**Coverage:** PT-sUSDe Morpho (F07-01), PT-weETH Morpho (F07-02), YT-weETH
points (F07-03), PT/SY redemption near maturity (F07-04).
**Insight:** F07-01 is the highest-magnitude *carry* in the corpus (+25-40% over
90d on $1M at K≈6); F07-03 is the highest-asymmetry (+30× base, -40% downside).
**Most likely empirically profitable:** **F07-01**.

### F08 — Ethena USDe / sUSDe
**Mechanism:** sUSDe APY funded by perp-basis; looped 4-5× on Morpho / Aave with
USDC debt = cleanest stable-stable carry.
**Coverage:** sUSDe-Morpho-USDC loop (F08-01), USDe peg arb (F08-02), PT-sUSDe
Morpho+Pendle flash (F08-03), sUSDe Aave stable-eMode loop (F08-04).
**Insight:** F08-01 and F08-04 represent the same trade on two MM venues — good
apples-to-apples for an Aave-vs-Morpho rate study.
**Most likely empirically profitable:** **F08-01** at $1M notional.

### F09 — Morpho Blue isolated markets
**Mechanism:** Morpho's free-flashloan callback enables atomic loop bootstrap;
per-market isolation supports 94.5% LLTV on assets Aave can't list.
**Coverage:** wstETH/WETH 94.5% loop (F09-01), sUSDe/DAI 91.5% loop (F09-02),
MetaMorpho idle-capture (F09-03), cross-market rate-arb (F09-04).
**Insight:** F09 is the strongest *mechanically-reproducible* family — every
strategy opens the position on-fork.
**Most likely empirically profitable:** **F09-02** at the Dec-2024 block.

### F10 — Aave v3 / Spark / GHO
**Mechanism:** Variable-rate GHO mint (discounted for stkAAVE holders) + Aave
eMode + Spark's DSR-pegged DAI = on-chain stable-rate basis surface.
**Coverage:** GHO + Balancer carry (F10-01), Spark sDAI eMode (F10-02),
Spark/Aave DAI rate arb (F10-03), GHO + stkAAVE + sDAI (F10-04).
**Insight:** F10-04 is the only strategy combining *safety-module yield* with
*discounted CDP mint*.
**Most likely empirically profitable:** **F10-02** in the high-DSR April-2024
window.

### F11 — Compound v3 + Fluid + Euler
**Mechanism:** USDC supply rates diverge 50-170 bps across Comet, Aave, Fluid,
Euler v2; Euler v2 EVC enables a same-tx multi-vault batch unique to it.
**Coverage:** Comet WETH loop (F11-01), Fluid wstETH/ETH smart collateral
(F11-02), Euler v2 EVC same-asset rate arb (F11-03), Comet ↔ Aave USDC arb
(F11-04).
**Insight:** F11-03 is the most-novel structural primitive — EVC's batch
semantics with sub-account collateral/controller separation can't be replicated
elsewhere.
**Most likely empirically profitable:** **F11-04** at any divergence block.

### F12 — Curve + Convex + bribes
**Mechanism:** vlCVX (16-week lock) generates snapshot-weighted gauge votes;
Votium / Hidden Hand pay bribes in arbitrary ERC20s.
**Coverage:** Convex frxETH/ETH (F12-01), vlCVX + Votium (F12-02), Convex
stETH/LDO triple-reward (F12-03), veCRV gauge-weight snipe (F12-04).
**Insight:** F12-02 is the highest cash-IRR positional trade in the corpus on a
per-CVX basis — gated only on willingness to lock 16 weeks.
**Most likely empirically profitable:** **F12-02** at the April-2024 Votium
round 56/57 boundary.

### F13 — Balancer / Uniswap v3 LP
**Mechanism:** Balancer's rate-provider cache (24h heartbeat) goes stale relative
to live LST `exchangeRate()`; UniV3 flash + 2-leg swap captures atomically.
**Coverage:** wstETH flash + Balancer arb (F13-01), rETH Balancer-vs-UniV3
(F13-02), Balancer composable stable LP (F13-03), UniV3 narrow LP (F13-04).
**Insight:** F13-01/02 are the same arb on two different LSTs and benefit most
from being **scanned across many blocks**.
**Most likely empirically profitable:** **F13-02** on a freshly-updated rETH
balance day.

### F14 — Synthetix atomic + sUSD
**Mechanism:** Synthetix's atomic exchange clamps to the worse-of-two oracles
(Chainlink + Uniswap TWAP); arbs the clamp when one oracle is stale.
**Coverage:** sETH/sUSD triangular (F14-01), sUSD depeg via atomic exit (F14-02),
sBTC/sETH/sUSD triangular (F14-03), post-Chainlink-update sandwich (F14-04).
**Insight:** F14-04 is event-driven (immediately after Chainlink updates) —
would benefit most from Wave-3 sweep automation.
**Most likely empirically profitable:** **F14-02** at the SVB weekend.

### F15 — EigenLayer native restake
**Mechanism:** Native EigenLayer captures EIGEN points without LRT wrapper risk;
cap-races and 7-day queues are exploitable secondary markets.
**Coverage:** stETH-direct vs ezETH (F15-01), cap-race keeper (F15-02), 7-day
queue secondary (F15-03), native-Eigen + Symbiotic dual-stack (F15-04).
**Insight:** F15-04 is the only strategy earning points from *two restake
protocols simultaneously on the same wstETH unit*.
**Most likely empirically profitable:** **F15-01** — deposit cash mechanics
are reproducible on-fork.

### F16 — Cross-CDP basis
**Mechanism:** Every CDP prices its borrow rate independently; basis between
GHO, crvUSD, DAI, LUSD borrow rates can exceed 100 bps.
**Coverage:** LUSD trove + Aave supply (F16-01), GHO vs crvUSD basis (F16-02),
DssFlash triangle GHO/crvUSD/DAI (F16-03), GHO mint → LUSD SP carry (F16-04).
**Insight:** F16-03's DssFlash triangle is the clearest expression of
"zero-capital, all-mechanics" — every leg goes through a canonical CDP primitive.
**Most likely empirically profitable:** **F16-04** at the pinned block
(~18-25% APR, heavily LQTY-priced).

### F17 — Yield-bearing stable carry
**Mechanism:** Each yield-bearing stable expresses yield differently (rebase,
share-price, wrapper) and trades at different Curve discounts.
**Coverage:** USDM rebase via Curve (F17-01), syrupUSDC vs sUSDS rotation
(F17-02), OETH/ETH atomic redeem (F17-03), OUSD via Aave + wOUSD (F17-04).
**Insight:** F17-03 is the clearest illustration of *why a permissionless redeem
path is structural alpha* — OETH has one and USDM/USDY don't.
**Most likely empirically profitable:** **F17-02** rotation at an entry-spread
block.

## 4. Cross-family observations

**Primitive concentration.** A small number of primitives dominate. Free
flashloans (Maker `DssFlash`, Morpho callback flash, Balancer Vault
`flashLoan`, UniV3 `flash`, Aave `flashLoanSimple`) appear in 11 strategies —
every atomic family relies on at least one. **Aave/Spark eMode** is the
single most-shared mechanism (33 README mentions across F01/F02/F04/F08/F10),
because elevated LTV is what turns 2-3% rate spreads into 10-20% leveraged
returns. **ERC4626 wrappers** (sDAI / sUSDS / sUSDe / syrupUSDC / wOUSD /
MetaMorpho / Spark vaults) appear in 15 strategies and form the "wrap-it-to-
use-it" bridge between rebasing yield and money-market collateral. **Curve**
is in 54 / 68 strategies — almost every stable-side mechanism touches it
somewhere.

**Mechanism combinations unique to this research.** Several pairings are
specifically Wave-2 novelties relative to standard "DEX arb / yield
aggregator" literature: stacked LRT re-hypothecation (Puffer → Karak →
Symbiotic), Pendle YT used as a *point-decoupling option* (F02-02 / F07-03),
Liquity v2 lowest-rate-trove redemption sniper, Euler v2 EVC multi-vault
same-tx batches, cross-CDP triangular arb GHO/crvUSD/DAI via DssFlash, and
Native-Eigen + Symbiotic dual-stack on a single wstETH unit.

**Empirical strength.** Wave-2 self-reported statuses cluster sharply: **F09**
is the only family where every strategy is `mechanically-reproducible` (Morpho
ABIs allow position open in one tx, marketId verifiable by keccak hash).
**F13** and **F17** also reach `mechanically-reproducible` in 2-3 of 4 strategies.
The weakest families are **F06 v2** (BOLD addresses TODO), **F15-03**
(secondary market does not exist on-chain at fork block, documented gap),
and several Pendle / Karak / Symbiotic vault addresses that are `TODO verify`
(§6). **F02 and F07 points trades** are theoretically the largest payoff but
are by construction off-chain (airdrop pricing).

**Gaps worth deepening.**
1. **Liquity v2.** Three of four F06 strategies depend on BOLD addresses
   not yet in `Mainnet.sol`. Once wired, F06-03/F06-04 become testable.
2. **Symbiotic / Karak vault registries.** F02-03 and F15-04 are gated on
   verifying canonical vault addresses at their fork blocks.
3. **Pendle YT modelling.** F02-02 and F07-03 self-report PnL ranges spanning
   4 orders of magnitude. A Wave-4 pass that models *point value
   distributions* explicitly would be valuable.
4. **MEV / private-orderflow assumptions.** F05-01, F05-02, F13-01, F13-02,
   F14-04 all assume a sole-searcher. A realistic Flashbots-competition
   haircut would compress the top-N rankings considerably.
5. **Atomic arb opportunity rate.** Many F03/F05/F08 atomic strategies are
   sized per-event; converting to $/30d requires modelling opportunity
   frequency from historical timeseries.

## 5. Verification checklist (for the user)

Wave-3 did **not** run `forge`. The user must verify each PoC.

```bash
cd /home/user/profit
cp .env.example .env && $EDITOR .env   # paste an archive RPC URL
make install                            # forge install forge-std + OZ
make build                              # compile interfaces + PoCs
make test-one ID=F01-01                 # run one strategy
forge test --match-path "strategies/**" -vv   # run all strategies
make summary                            # aggregate the printed PnL blocks
```

### Per-strategy RPC requirements

| Strategy ID(s) | RPC requirement |
| -------------- | --------------- |
| **All others (≈58)** | Generic mainnet archive RPC (`RPC_URL` in `.env`). Standard providers (Alchemy, QuickNode, Infura) work. |
| F01-04, F03-01, F03-04, F04-01, F06-01, F06-02, F14-02, F14-03 | **Older blocks (pre-2024 / pre-2023)** — verify provider supports the requested archive depth. |
| F03-02 (Renzo depeg), F03-03 (rETH oracle update), F12-02 (Votium round boundary), F13-02 (rETH oracle update), F14-04 (Chainlink update tail) | Block must be **event-pinned**; the README states the canonical block but Wave-3 sweep across nearby blocks is recommended. |
| F06-03, F06-04 (Liquity v2 BOLD), F02-03 (Karak/Symbiotic), F15-04 (Symbiotic) | Requires **verifying TODO-flagged addresses** at the fork block before running (§6). |

All other strategies need only the standard `RPC_URL` set in `.env`.
